01 counting systems

The same number, three costumes

A "base" is just how many digits you count with before rolling over. We use base 10 (0–9) because we have ten fingers. Computers use base 2 (0–1) because a wire is either on or off. And base 16 (0–9, then A–F) is a shorthand humans use for binary. Type a number in any box — the others follow instantly, and the breakdown shows why.

binary base 2
decimal base 10

offensive angleAttackers abuse base-switching to dodge filters: the IP 127.0.0.1 can be written as 0x7f000001 (hex) or 2130706433 (decimal) — same address, unrecognisable to a naive blocklist. Seeing through bases is how you catch obfuscated IPs and URLs.

02 units of data

Bits, bytes, and the trap in between

A bit is one 0-or-1. Eight bits make a byte — roughly one letter. From there the units climb ×1000 each step. The famous gotcha: internet speed is measured in bits (Mbps) but files in bytes (MB) — an 8× difference that confuses everyone.

the bits-vs-bytes trap — try it
Mbps (megabits/sec) ÷ 8 = 12.5 MB/s (megabytes/sec)
why a security person caresData exfiltration is measured in bytes — spotting an unusual volume of outbound data is a classic detection. And confusing bits with bytes (the lowercase b vs uppercase B) causes real misreads of network graphs and tool output.

// note: storage vendors use 1000 (GB), but your OS often counts in 1024 (GiB) — which is why a "1 TB" drive shows as ~931 GB. Same bytes, different rounding.

03 text as numbers

ASCII → Unicode → UTF-8

ASCII gave the first 128 characters a number (A = 65) — but only English. Unicode fixed that: one giant table assigning every character in every language (and every emoji) a number called a code point (written U+XXXX). UTF-8 is how those code points get stored as bytes — English stays 1 byte, but other scripts and emoji take 2, 3, or 4. Type anything below — your name, Urdu, an emoji — and watch it become bytes.

why a security person caresBecause different characters can look identical but have different code points, attackers register lookalike domains (see §05). Analysts must normalise text to one form before comparing it — otherwise a check for "admin" can be bypassed with a lookalike character. The 1-byte backward compatibility of UTF-8 with ASCII is also why old tools still mostly work.

04 binary-safe text

Base64: dressing bytes as plain text

Some channels — email bodies, URLs, JSON — only cope with plain text. Base64 repackages any bytes into a safe alphabet (A–Z, a–z, 0–9, + /) by taking 3 bytes at a time and re-slicing them into 4 six-bit chunks. It's everywhere. Encode and decode live below.

→ base64
how to recognise Base64 on sight:
› only A–Z a–z 0–9 + /   › length is a multiple of 4   › often ends in = or ==   › ~33% longer than the original
offensive angleAttackers Base64-encode commands to slip past text filters — the infamous powershell -enc <base64> hides the real command from a casual glance. But it's not encryption — it reverses instantly, so a defender who spots a Base64 blob in a log just decodes it to reveal exactly what was attempted.

// verify: echo -n "Man" | base64 → TWFu · echo -n "TWFu" | base64 -d → Man

05 the web's costume — and an attack

URL encoding, and lookalike domains

URLs can't contain spaces or many symbols, so they get percent-encoded: each forbidden byte becomes % followed by its hex value (space → %20). Type below and watch it happen — the changed parts glow.

attack characters analysts watch for, url-encoded:
%27 = ' (SQL injection)   %3C = < (XSS)   %2F = /   %00 = null byte
and the bypass trick — double encoding: < → %3C → %253C (a filter blocking "%3C" misses "%253C", which the server may decode back to "<").

Now the payoff — where representation becomes a real attack. These two domains look identical. One is real; one uses a Cyrillic letter that looks like a Latin one. Click "inspect" to unmask the fake.

A "homograph attack" hides a lookalike character in a familiar name. To your eye they match. To the computer — which sees code points, not shapes — they're completely different addresses.
offensive & defensiveOffense: register a lookalike domain to phish credentials — the victim reads "apple.com" and trusts it. Defense: browsers convert such domains to punycode (that xn-- prefix) to expose them, and analysts normalise and compare code points, never shapes. This is data representation as a live attack surface.
