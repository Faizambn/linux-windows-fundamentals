Active Directory is the phone book and the security guard for almost every corporate network — it decides who every user is and what they can touch. Compromise it and you own the whole company. That's why it's the #1 target in enterprise attacks.

01 the org chart of the network

Structure & objects

AD organises a company into a hierarchy. A forest is the top-level security boundary; inside it are domains; inside those, Organizational Units (OUs) group the objects — users, computers, and groups. The Domain Controller is the server running it all. Click any part to learn it.

Click an item on the left to explore the structure.
why it's the top targetEverything hangs off the Domain Controller — it holds the NTDS.dit database with every user's password hash. Own a DC and you own every account. That's why attackers' entire goal in a corporate breach is usually "get to Domain Admin."

02 how you prove who you are

Kerberos authentication

When you log into a domain, you don't send your password to every server. Instead Kerberos uses tickets. You prove yourself once to the KDC (the Domain Controller) and get a TGT (Ticket-Granting Ticket); you then trade that for service tickets to access specific resources. Step through it below.

👤 Client
alice
🏛️ KDC
Domain Controller
🖥️ Service
FileServer
1. AS-REQ → Client proves identity to the KDC (timestamp encrypted with her password hash)
← 2. AS-REP KDC returns a TGT — an "I'm authenticated" pass, sealed with the secret krbtgt key
3. TGS-REQ → Client shows the TGT and asks: "let me reach FileServer"
← 4. TGS-REP KDC returns a Service Ticket for FileServer
5. AP-REQ → Client presents the Service Ticket to FileServer → access granted
Press play to watch alice authenticate and reach the file server.
attack & defenseTickets are exactly what attackers steal or forge. Kerberoasting requests service tickets and cracks them offline for the service account's password. A Golden Ticket forges an unlimited TGT using the stolen krbtgt key — total domain control. (The older NTLM protocol, still around as fallback, enables Pass-the-Hash.) Defenders watch Kerberos events 4768/4769 and use strong service-account passwords.

// TGT = your master pass · Service Ticket = a day-pass for one resource · KDC = the ticket office, run by the Domain Controller.

03 from nobody to domain admin

Attack paths

Attackers rarely go straight to Domain Admin. They follow a path — a chain of "this user is a member of that group, which has admin rights on that machine, where an admin is logged in…" — hopping link by link. Trace one below.

👤 alice
a normal, low-privilege user (the foothold)
↓ is a member of
👥 IT-Support group
a group alice belongs to
↓ has local admin on
🖥️ SVR-01
a server IT-Support can administer
↓ where a session exists for
👤 bob (Domain Admin)
bob is logged in — his token can be stolen
↓ is a member of
👑 Domain Admins → the Domain
game over — full control
Press "trace the path" to follow the chain from a normal user to full domain control.
the key tool — BloodHoundThis graph is exactly what BloodHound does: it collects AD data (users, groups, sessions, admin rights) and draws it as a graph, then finds the shortest path from any user to Domain Admin. Attackers use it to plan; defenders use it to find and cut these paths before attackers walk them.
