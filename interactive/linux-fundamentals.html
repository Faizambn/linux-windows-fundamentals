attack & defensePermissions are the #1 Linux privilege-escalation surface. A world-writable file others can edit, or a SUID binary owned by root (it runs as root no matter who launches it), can hand an attacker full control. Defenders hunt for these: find / -perm -4000 lists every SUID file on the system.

04 the SOC superpower

Finding an attack in a log

The real power of Linux is chaining small tools with the pipe (|) — the output of one becomes the input of the next. Here's the exact skill a SOC analyst uses: taking a raw auth log and, step by step, pulling out a brute-force attacker. Click each stage to build the pipeline.

attack & defenseAn attacker brute-forcing SSH leaves hundreds of "Failed password" lines. In four piped commands you go from a wall of raw log to "one IP tried 312 times" — a confirmed attack, ready to block. This is genuinely what Tier-1 SOC work looks like.

// the real one-liner: grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr
