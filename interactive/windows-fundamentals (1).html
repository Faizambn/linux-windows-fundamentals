01 the system's brain

The Windows Registry

The Registry is a giant hierarchical database of settings for Windows and its programs — think of it as Linux's /etc, but as one searchable tree. It's organised into hives (top-level branches), then keys (folders) holding values (settings). Click around — the ⭐ entries are the ones every analyst watches.

Click a hive or key on the left.
attack & defenseThe Registry is the #1 place malware hides to survive a reboot (persistence). The "Run" keys auto-launch programs at login — so attackers add themselves there. Defenders (and forensics tools) inspect these exact keys first. It's also where password hashes (SAM) and huge amounts of forensic evidence live.

// open it yourself: press Win+R → type regedit · or query from CMD: reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Run"

02 the blue-team core

Event Logs & Event IDs

Windows records almost everything as numbered events in the Event Log. Learning the key Event IDs is the heart of SOC work — they're how you reconstruct what happened. Click an ID to learn it, then hit "replay an attack" to watch a real intrusion tell its story through the logs.

Click an Event ID above to learn what it means.
why a security person caresThis is the job. A burst of 4625 (failed logons) then a 4624 (success) = a brute-force that worked. A 4720 (new user) you didn't expect = the attacker creating a backdoor account. A 1102 (log cleared) = someone covering their tracks. You read these all day.

// open it yourself: Win+R → eventvwr → Windows Logs → Security

03 identity & privilege

Users, SIDs & UAC

Windows identifies every account not by name but by a SID (Security Identifier) — a unique string ending in a RID. Some RIDs are famous: 500 is always the real Administrator. And UAC is why even an admin runs with reduced power until they explicitly elevate.

You're using the standard token. Click below to request elevation.
attack & defenseAttackers aim for the RID-500 admin or SYSTEM (the most powerful account of all). UAC bypasses are techniques to grab the elevated token without the prompt, and token theft/impersonation steals another logged-in user's access. Defenders monitor for elevation events (4672) and unusual privilege use.

// see your identity: whoami /user (shows your SID) · whoami /priv (your privileges)

PowerShell & CMD

attack & defensePowerShell is a double-edged sword: admins love it, and so do attackers ("living off the land" — using built-in tools so nothing looks out of place). That's why defenders enable PowerShell script-block logging, which records the real commands even when they're obfuscated or Base64-encoded.

05 the lay of the land

Filesystem & the defender's toolkit

Unlike Linux's single / tree, Windows splits storage across drive letters (C:\, D:\). Knowing which folders hold what — and which ones malware loves — is core to spotting something out of place. Windows also ships in editions (Home, Pro, Enterprise) built on the same core. Click a location to learn it.

Click a location on the left.
the defender's toolkit — SysinternalsMicrosoft's free Sysinternals suite is the Windows investigator's essential kit: Process Explorer (a super Task Manager showing parent/child and signatures), Autoruns (lists everything set to auto-start — the fastest way to spot persistence), and Procmon (records every file/registry/process action live). If you learn one Windows toolset, learn this one.

// grab it free: learn.microsoft.com/sysinternals · Autoruns alone will teach you more about persistence than any lecture.
